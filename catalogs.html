JBang supports creating and sharing catalogs. When shared on github, gitlab, or bitbucket, implicit aliasing allows to run scripts from remote repositories.

As described in jbang documentation, executing `jbang hello@acme` will run `hello` alias found in `acme/jbang-catalog/jbang-catalog.json` of the default branch searched on github, gitlab and bitbucket in that order.

Following table attempts to consolidate aliases from public repositories on github.

To execute any alias, run `jbang <>`. Example - jbang quarkus-starter@38leinaD.

NOTE: Always verify any script code before executing it.
